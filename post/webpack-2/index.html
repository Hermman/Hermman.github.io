多页应用

// webpack.config.js
const path = require("path")
const HtmlWebpackPlugin = requre("html-webpack-plugin")

  mode: "development",

    home: "./src/home.js",

配置source-map (源码映射)

源码映射 会单独生成一个 sourcemap 文件，出错了，会标识当前报错的行和列。devtool 配置字段 ，添加映射文件帮助我们调试源代码。

文件配置如下：

module.exports = {
  // source-map： 生成一个新的.map源文件，报错会产生行和列
  devtool: "source-map"
    
  // eval-source-map: 打包时不会生成一个单独的映射文件，但可以显示行和列
  devtool: "eval-source-map"
    
  // cheap-module-source-map:不会产生列，但会生成一个单独的映射文件
  devtool: "cheap-module-source-map"
    
   // 不会生成文件，集成在打包后的文件中，不会产生列
   devtool: "cheap-module-eval-source-map"
}

区分不同的环境

在开发中，webpack打包配置一般都有两个环境配置（development 和 production）。所以打包配置文件一般会有三个：

  • webpack.base.js

    基础打包配置：打包入口、出口文件、loader等

  • webpack.dev.js

    开发环境打包配置：devServer、devtool 等

  • webpack.prod.js

    生产环境打包配置：代码压缩，性能优化配置等

用法：使用 webpack-merge

  • 安装： yarn add webpack-merge -D

  • 配置：

    // webpack.base.js
const path = require("path");
const { CleanWebpackPlugin } = require("clean-webpack-plugin");
const HtmlWepackPlugin = require("html-webpak-plugin")

module.exports = {
  entry: {
      app: "./src/index.js"
  },
  output: {
    path: path.resolve(__dirname, "dist"),
    filename: [name].bundle.js
  },
   module: {
   	relus: [
      { 
    	test: /\.js/, 
    	use: {
    	  loader: "babel-loader"
		} 
	   }
    ]
  },
  plugins: [
    new CleanWebpackPlugin('dist'),
    new HtmlWepackPlugin({
      template: "./public/index.html",
      filename: "index.html"
    })
  ]
}

    // webpack.prod.js
const merge = require("webpack-merge");
const base = require("./webpack.base");

module.exports = merge(base, {
  mode: "production",
});


    // webpack.dev.js
const merge = require("webpack-merge");
const base = require("./webpack.base");

module.exports = merge(base, {
  mode: "development",
  devtool: "source-map",
  devServer: {
    port: 3000,
    contentBase: "./dist",
  },
});

    // package.json
"scripts": {
  "start": "webpack-dev-server --open --config webpack.dev.js",  
  "build": "webpack --config webpack.prod.js"
    },

webpack 抽离公共代码

代码配置如下：

module.exports = {
  // 优化项
  optimization: {
    splitChunks: {  // 代码分割
      cacheGroups: { // 缓存组
        common: {    // 公共的模块抽离
          chunks: "initial",
          minSize: 0,  // 字节大小
          miniChunks: 2,  // 最小被引入次数
        },
          
        // 第三方模块抽离
        vendor: {
          priority: 1,  // 权重，代码抽离时优先级高
          test: /node_modules/,  // 只匹配正则里的
          chunks: "initial",
          minSize: 0,
          minChunks: 2,
        }
      }
    }
  }
}

懒加载

懒加载或者按需加载，是一种很好的优化网页或应用的方式。这种方式实际上是先把你的代码在一些逻辑断点处分离开，然后在一些代码块中完成某些操作后，立即引用或即将引用另外一些新的代码块。这样加快了应用的初始加载速度，减轻了它的总体体积，因为某些代码块可能永远不会被加载

热更新

  • 第一步 在devServer中配置：

    devServer: {
  hot: true, // 启用热更新
  port: 3000,
  open: true,
  contentBase: './dist'
}

  • 第二步 在plugin 插件数组中引用webpack 内置热更新插件

    plugins: [
  new webpack.HotModuleReplacementPlugin(),
  new webpack.NameModulesPlugin(), // 可指定哪个模块人更新
]
