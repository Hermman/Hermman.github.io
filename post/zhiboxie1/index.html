常见的三大直播协议

· 2020-07-27 · # 直播协议

受到大环境影响，越来越多的公司开始研发自己的直播平台，作为开发者，有必要了解或学习，那就先从直播协议开始吧！

常见的直播协议

国内常见的直播协议有：RTMP、HLS、HTTP-FLV

RTMP

RTMP, 全称 Real Time Messageing Protocol，实时消息传送协议。是 Adobe 公司为 Flash 播放器和服务器之间音视频数据传输开发的私有协议。

工作原理

工作在TCP之上的明文协议，默认端口是 1935 。协议中的基本数据单元为消息；传输过程中消息会被拆分为更小的消息块（chunk）单元，最后将分割的消息块通过TCP协议传输，接收端再反解 接收的消息块恢复成流媒体。

优点

  1. RTMP的延迟相对比较低，一般延迟在1~3s之间；
  2. 对所有支持Adobe Flash的设备都支持RTMP
  3. 适合长时间播放

缺点

  1. 它是基于 TCP 传输的，非公共端口，可能会被防火墙阻拦
  2. 在IOS端支持不友好

FLV

FLV，是Adobe公司推出的另一种视频格式，是一种数据存储容器格式。

其格式相对简单轻量，加载速度极快，文件后缀名为 .flv 。

原理

HTTP-FLV 是将流媒体封装成 FLV 格式，然后通过 HTTP协议传输给客户端。

HTTP-FLV 依靠 MIME 的特性，根据协议中的 Content-Type 来选择相应的程序去处理相应的内容，使得流媒体可以通过 HTTP 传输

优点

  1. 相比RTMP协议，HTTP-FLV 能够很好的穿透防火墙，它是基于 HTTP/80传输，可以有效的避免防火墙的拦截。
  2. 支持使用 HTTPS 加密传输，能够兼容 Android 、ios 的移动端。

缺点

  1. 由于它的传输特性，会让流媒体资源缓存在本地客户端，在保密性方面不够好
  2. 因为网络流量较大，它也不适合做拉流协议

HLS

HLS，是苹果公司基于HTTP的流媒体传输协议。主要应用于ios设备。

工作原理

不同于其他协议的，它会在服务端将流媒体数据切割成连续的时长较短的ts小文件，并通过M3U8索引文件按序访问ts文件，客户端只要不停放的按序播放从服务端获取到的文件，从而实现播放音视频。

HLS的优势：

  • Apple的全系列产品的支持；
  • 穿透防火墙，基于http/80传输，有效避免防火墙拦截
  • 性能高，通过http传输，支持网络分发，CDN支持良好，且自带多码率自适应

HLS的劣势:

  • 时效性差，延迟高

  • 文件碎片，造成海量的小文件，对存储和缓存有一定的挑战性
